# Models

This folder documents the trained models used in the Arabic NLP project.

The actual trained model folders are not included in this repository because they are large and may exceed GitHub file size limits. They can be regenerated by running the notebooks or placed locally when running the Streamlit application.

---

## Model Tasks

This project uses models for two main NLP tasks:

1. **Arabic Question Classification**
2. **Arabic Question Answering**

The classification model predicts the category of the Arabic question, while the question-answering model generates an Arabic answer for the user question.

---

# 1. Arabic Question Classification Model

The classification part of the project predicts the category of an Arabic question.

The final Streamlit app expects the trained classification model to be stored locally in this folder name:

```text
arabert_classification_model/
```

This model is used to classify Arabic questions into predefined categories such as education, technology, health, science, politics, sports, culture, religion, and other topics.

---

## Classification Categories

The classification model predicts one of the following Arabic categories:

* الاقتصاد والعمل
* البيئة والطاقة
* البيولوجيا
* التاريخ
* الترفيه
* التطوع
* التعليم
* التكنولوجيا
* الثقافة
* الجغرافيا
* الدين
* الرياضة
* السفر والسياحة
* السياسة والقانون
* الصحة
* العلوم
* علم الاجتماع

---

## Classification Model Purpose

The classification model is used to:

* Understand the topic of the user question.
* Assign the question to the most suitable category.
* Support the Streamlit app by displaying the predicted category.
* Organize Arabic questions based on their meaning and subject.

---

# 2. Arabic Question Answering Model

The question-answering part of the project generates Arabic answers for Arabic user questions.

The final Streamlit app expects the trained QA model to be stored locally in this folder name:

```text
qwen_qa_model/
```

This model is used to generate Arabic answers based on the input question.

---

## QA Model Purpose

The QA model is used to:

* Receive an Arabic question from the user.
* Generate a direct Arabic answer.
* Support the Arabic NLP system inside the Streamlit app.
* Provide the main question-answering functionality of the project.

---

# 3. Models Used by the Streamlit App

The Streamlit application uses two local model folders:

```text
arabert_classification_model/
qwen_qa_model/
```

The app workflow is:

1. The user enters an Arabic question.
2. The QA model generates an Arabic answer.
3. The classification model predicts the question category.
4. The app displays the original question, generated answer, predicted category, and translated output.

---

# 4. Expected Local Project Structure

To run the Streamlit app locally, place the trained model folders in the same directory as `app.py`.

Expected structure:

```text
arabic-nlp-question-answering-classification/
│
├── app.py
├── arabert_classification_model/
├── qwen_qa_model/
│
├── models/
│   └── README.md
│
└── requirements.txt
```

---

# 5. Why Models Are Not Uploaded

The trained model folders are not uploaded to GitHub because they may contain large files, such as:

* model weights
* tokenizer files
* configuration files
* checkpoint files

Uploading these files can make the repository too large and may exceed GitHub file size limits.

Instead, this README explains which model folders are needed and where they should be placed.

---

# 6. How to Regenerate the Models

The models can be regenerated by running the project notebooks.

| Notebook                               | Purpose                                                    |
| -------------------------------------- | ---------------------------------------------------------- |
| `arabic_question_classification.ipynb` | Trains and saves the Arabic question classification model. |
| `arabic_question_answering.ipynb`      | Trains and saves the Arabic question-answering model.      |

After running the notebooks, save the final models using these folder names:

```text
arabert_classification_model/
qwen_qa_model/
```

These names are important because the Streamlit app loads the models using these exact folder paths.

---

# 7. Important Notes

* The model folders must exist locally for the Streamlit app to run.
* The model folders are not included in this repository to keep it lightweight.
* API keys should not be stored inside the code.
* Use Streamlit secrets or environment variables for external API keys.
* Results and model comparisons are discussed in the main project README.
